# Duplicates and conflicts

## Exact duplicates

- Every one of the 14 substantive `files.zip` members is byte-for-byte identical to the corresponding expanded file under `files/`. The ZIP also contains 15 macOS `__MACOSX`/AppleDouble metadata entries with no analytical value.
- No two separate expanded physical files have the same SHA-256 hash.
- `02_outputs_explained.docx` and `02_outputs_explained.md` cover substantially the same explanatory material but are not exact byte/content duplicates; the DOCX has additional appended claims.

## Near-duplicate notebooks

- `files/EV_Bus_Text_Analysis.ipynb` is the unexecuted 9-code-cell template; `files/EV_Bus_Text_Analysis-2.ipynb` is its executed 10-code-cell extension with the camp comparison. Their core setup, cleaning, word cloud, VADER, LDA and pyLDAvis code is effectively the same.
- `02_sentiment_topic_analysis.ipynb` overlaps the same cleaning/sentiment/LDA functions but belongs to a different, missing custom project architecture and adds lifecycle/OEM-group logic. It should not be assumed interchangeable because its schemas, grouping, preprocessing and data provenance differ.

## Conflicting datasets and outputs

- **Tender conflict:** `ev_bus_tenders_ORIGINAL_manus.csv` assigns city lots to OEMs and supplies estimated L1 rates; `ev_bus_tenders_CORRECTED.csv` states city-to-OEM mapping is not disclosed and separates four aggregate OEM awards from five city allocations. The corrected table supersedes the original.
- **Market-share conflict:** `oem_benchmark.csv` market-share percentages and cumulative order counts do not reconcile to a common market denominator, as already flagged in `PROJECT_HANDOFF.md`.
- **CAGR definition conflict:** proposal/analysis material uses roughly 86% for unit/volume growth while market-policy material uses roughly 21% for revenue growth. These are different measures, not interchangeable market-growth estimates.
- **Corpus lineage conflict:** `merge_corpus.py` claims per-file uniform stamping and outputs `source_file`, but the supplied `ev_bus_text_corpus.csv` instead has row-level brands/sources, URL/date fields and no `source_file`. The handoff confirms the exact final merge was interactive, so the script is not a fully reproducible generator.
- **Instagram interpretation conflict:** the raw file is a post scraper dominated by brand captions, while some documentation loosely describes Instagram comments. Audience comments exist only in nested `firstComment/latestComments` fields; the dataset is not a dedicated comment scrape.
- **Text-result conflict:** the synthetic custom notebook reports 35-row lifecycle/OEM results; the real-corpus notebook reports 548-row VADER/LDA/camp results. They answer related but different questions and must never be combined as one run.
- **Document caveat conflict:** `02_outputs_explained.docx` clearly says its numeric outputs are synthetic, but its final appended â€śConnecting sentiment with topicsâ€ť claims (for example, â€śPost-use comments are 48% negativeâ€ť) are presented without a repeated synthetic label and are not generated by the visible notebook cells.

## Synthetic or unsupported results that could be mistaken for findings

- All saved outputs in `02_sentiment_topic_analysis.ipynb` are from `sample_ev_bus_comments.csv`; displayed URLs explicitly use `synthetic://`.
- The lifecycle counts, 20/9/6 sentiment counts, OEM-stage percentages and topic coherence from that notebook are demonstrations only.
- The appended conclusions in `02_outputs_explained.docx` can be mistaken for empirical results despite the opening synthetic warning.
- `ev_bus_tenders_ORIGINAL_manus.csv` contains known fabricated/unsupported city-to-OEM mappings and estimated rates.
- `market_policy_facts.csv` and `adoption_barriers.csv` each contain a row whose `source_url` is `https://pasted_content_txt (user input)`, which is not evidence.
- `ebus_opinions.csv` is an AI/Manus-compiled quote table. It may contain real excerpts and URLs, but the repository lacks collection logs or snapshots sufficient to authenticate each quote.
- The nine-page strategic DOCX contains research conclusions but no embedded data appendix or executable lineage; treat as analysis output, not raw evidence.
